# Cloudinary Integration

Cloudinary handles all media in Locafy: product images, shop banners/logos, vendor reel videos, and user avatars. It provides image optimization, on-the-fly transformations, and adaptive HLS video streaming.

---

## Setup

1. Create a free account at [cloudinary.com](https://cloudinary.com)
2. Dashboard → Settings → API Keys
3. Copy **Cloud Name**, **API Key**, and **API Secret**

---

## Spring Boot Configuration

```java
// CloudinaryConfig.java
@Configuration
public class CloudinaryConfig {

    @Bean
    public Cloudinary cloudinary(
            @Value("${cloudinary.cloud-name}") String cloudName,
            @Value("${cloudinary.api-key}") String apiKey,
            @Value("${cloudinary.api-secret}") String apiSecret) {

        return new Cloudinary(ObjectUtils.asMap(
            "cloud_name", cloudName,
            "api_key", apiKey,
            "api_secret", apiSecret,
            "secure", true
        ));
    }
}
```

## Cloudinary Service (`CloudinaryService.java`)

```java
@Service
@RequiredArgsConstructor
public class CloudinaryService {

    private final Cloudinary cloudinary;

    // Upload product image
    public String uploadImage(MultipartFile file, String folder) throws IOException {
        Map<String, Object> params = ObjectUtils.asMap(
            "folder", folder,
            "resource_type", "image",
            "format", "webp",           // auto-convert to WebP
            "quality", "auto:good",      // smart quality
            "fetch_format", "auto"
        );
        Map result = cloudinary.uploader().upload(file.getBytes(), params);
        return (String) result.get("secure_url");
    }

    // Upload reel video with HLS transcoding
    public Map uploadReel(MultipartFile file, String vendorId) throws IOException {
        Map<String, Object> params = ObjectUtils.asMap(
            "resource_type", "video",
            "folder", "reels/" + vendorId,
            "eager", new Map[]{ ObjectUtils.asMap(
                "streaming_profile", "hd",
                "format", "m3u8"                // HLS master playlist
            )},
            "eager_async", true,
            "notification_url", "https://api.locafy.in/api/reels/cloudinary-notify"
        );
        Map result = cloudinary.uploader().upload(file.getBytes(), params);
        return result;
    }

    // Delete an asset
    public void delete(String publicId, String resourceType) throws Exception {
        cloudinary.uploader().destroy(publicId,
            ObjectUtils.asMap("resource_type", resourceType));
    }
}
```

## Image Transformations

Cloudinary transformations are applied via URL parameters — no backend processing needed.

### Product Image (thumbnail)

```
https://res.cloudinary.com/locafy/image/upload/w_400,h_400,c_fill,f_webp,q_auto/products/product123.jpg
```

| Parameter | Meaning |
|-----------|---------|
| `w_400,h_400` | Resize to 400×400 |
| `c_fill` | Crop to fill (centre crop) |
| `f_webp` | Convert to WebP for smaller size |
| `q_auto` | Cloudinary picks optimal quality |

### Shop Cover Banner

```
https://res.cloudinary.com/locafy/image/upload/w_1200,h_400,c_fill,f_webp,q_auto/shops/shop123_cover.jpg
```

### Shop Card Thumbnail

```
https://res.cloudinary.com/locafy/image/upload/w_300,h_200,c_fill,f_webp,q_auto/shops/shop123_cover.jpg
```

### Avatar (Circle crop)

```
https://res.cloudinary.com/locafy/image/upload/w_100,h_100,c_thumb,g_face,r_max,f_webp/avatars/user123.jpg
```

## Video — Adaptive HLS for Reels

When a vendor uploads a video, Cloudinary automatically creates a multi-bitrate HLS stream:

**Profiles generated by `streaming_profile: hd`:**
- 360p (mobile, low bandwidth)
- 480p
- 720p (HD)
- 1080p (Full HD, if source allows)

**HLS master playlist URL format:**

```
https://res.cloudinary.com/locafy/video/upload/sp_hd/reels/vendor123/reel456.m3u8
```

This URL is stored in the `reels.videoUrl` field and loaded by HLS.js in the frontend player.

**Auto-generated thumbnail:**

```
https://res.cloudinary.com/locafy/video/upload/so_0,w_400,h_711,c_fill/reels/vendor123/reel456.jpg
```

`so_0` = snapshot at 0 seconds.

## Reel Webhook (Transcoding Complete)

When HLS transcoding finishes (async), Cloudinary POSTs to our webhook:

```java
@PostMapping("/api/reels/cloudinary-notify")
public ResponseEntity<Void> cloudinaryNotify(@RequestBody CloudinaryNotifyDto dto) {
    if ("eager".equals(dto.getNotificationType()) && "complete".equals(dto.getStatus())) {
        String publicId = dto.getPublicId();
        String hlsUrl = dto.getEager().get(0).getSecureUrl();
        String thumbnailUrl = cloudinaryService.getVideoThumbnail(publicId);

        reelService.markReady(publicId, hlsUrl, thumbnailUrl);
    }
    return ResponseEntity.ok().build();
}
```

Until the reel is marked ready, it is saved as a draft and not shown in the feed.

## Upload Approach

All file uploads go through the **Spring Boot backend** — never directly from the browser to Cloudinary. This ensures:
- File validation (type, size) happens server-side
- Cloudinary credentials are never exposed to the client
- Files are associated with the correct authenticated user
